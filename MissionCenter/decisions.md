# 決策紀錄

- 2026-07-17：0.4.7 定位為信任與穩定性版本；先修正文件、隱私、資料清除、長文本與 MV3 可維護性，再加入 Meta AI Beta。
- 2026-07-17：Meta AI 以預設未啟用的 Beta provider 加入，不改變既有四家預設；自動測試通過不等同登入態實機驗收。
- 2026-07-17：Provider selector 必須隨擴充套件封裝，不採遠端 selector 設定；MV3 不使用 `chrome.alarms` 假裝保活，改以持久化 checkpoint 與明確中斷診斷補強。
- 2026-07-17：本機最新辯論狀態採 24 小時保留上限，並提供明確的一鍵清除入口。
- 2026-07-17：Provider 錯誤分成可恢復與不可恢復：服務超載最多自動重新整理並重送 3 次；額度／使用上限不重試。最終失敗時，把網頁可見原文標成「服務狀態」提供其他 AI 閱讀，但不計為正式回答。
- 2026-07-10：抓內鬼模式保留單一 UI 選項，但內部升級為高級規則：每局 0 或 1 位內鬼、內鬼任務改為用半真半假與定義/判準偷換讓討論偏航、第一輪互評禁止指認，最後一輪才允許判斷「沒有內鬼」或「誰最像內鬼」。原因是舊規則「必定有內鬼 + 第一輪要求抓人 + 明顯破綻」導致模型開局秒殺。
- 2026-06-19：MV3 service worker 重啟時只恢復 `waiting_for_user` 的完整 DebateEngine 快照；`running` 狀態標示中斷並要求重試，避免從不可重入的網頁自動化中段續跑。
- 2026-06-19：所有 provider 非同步工作以 generation token 防止 stale write；stop/reset/new run 會使舊 token 失效。
- 2026-06-19：Gemini 採「按最近送出鍵、觀察 4 秒、必要時只補一次 Enter」策略；其他 provider 保留原送出行為。
- 2026-06-20：基於合併後的 CodeRabbit follow-up 修正與測試補強，商店候選版號升為 0.4.1，避免重新上傳仍沿用已驗證完的 0.4.0 號碼。

- 2026-06-18：本次審查以 `d12bca6`（0.4.0 RC 版本點）到 `HEAD` 為範圍，先找功能回歸與缺測，不夾帶無關重構。
- 2026-06-05：快速鬥技場採用「先逐家啟用並送出，再逐家啟用收回」的 carousel scheduler，而不是完全背景並行。原因是 AI 網頁分頁在背景可能被 Chrome 節流，穩定性比理論極速更重要。
- 2026-06-05：移除 Mock Mode，因為目前專案已進入真實網頁自動化試玩階段，Mock 入口容易讓 UI 變複雜。
- 2026-06-05：總結辯論以目前 active provider tab 作為來源與最後裁判，其他勾選 provider 負責第一輪與互評。
- 2026-06-05：CodeRabbit review 暫停到主人明確同意第三方資料傳送後再執行。
- 2026-06-11：Free 版保留 ChatGPT/Gemini/Grok/Claude 四家 AI 基礎輪流辯論；快速鬥技場與總結辯論先列為 Pro 預留入口，不接付款後端。
- 2026-06-11：上架前 package zip 只收 manifest、assets、src；store listing、privacy policy、screenshot checklist 進 repo，但不打進商店 zip。
- 2026-06-14：快速鬥技場與總結辯論不再放外層兩顆醒目按鈕，改成進階辯論設定裡的互斥模式選項；外層主按鈕依選項切換行為。
- 2026-06-14：依主人要求，五連點 Free badge 的 Pro 解鎖保留為公開彩蛋並打進 Chrome Web Store zip。
- 2026-06-14：辯論輪次代表交叉評析輪數，第一輪回答固定一次；支援 1 到 5 輪，最後總結會彙整所有互評輪。
- 2026-06-14：提詞先維持繁體中文並強化輪次標籤；若日後要英文運行，應做成明確的語言模式，而不是半套混用。
- 2026-06-14：上架候選版本升為 0.4.0，避免沿用 0.3.0 造成 Chrome Web Store 更新版本號衝突。
